DEMO 01

The Backstage Pass: locals()

How Python remembers your variables — revealed live
locals_demo.py
def my_func():
    a = 10
    b = "hello"
    c = [1, 2, 3]
    print(locals())

my_func()
▶ Output
{'a': 10, 'b': 'hello', 'c': [1, 2, 3]}
locals_vs_globals.py
global_var = "I'm global"

def demo():
    local_var = "I'm local"
    print("locals:", locals())
    print("global?", "global_var" in globals())

demo()
▶ Output
locals: {'local_var': 'I'm local'}
global? True
🔒 Local Scope
Each function gets its own private memory bubble. Variables born when called, erased when the function returns.
📖 Returns a Dictionary
locals() returns a live snapshot — every variable name mapped to its current value inside that function.
🎯 Why It Matters
Proves Python is tracking everything inside your function as a dictionary. globals() shows the master ledger — every variable available in the entire script.
DEMO 02 — COUNTER-INTUITIVE!

The Empty List Surprise: any() & all()

What happens when you pass an empty list? The answer will surprise you.
any_demo.py
print(any([0, 0, 1, 0]))  # has a 1 ✓
print(any([0, 0, 0]))     # all zeros
print(any([]))           # ← empty list??
print(any([None, False, 0, "non-empty"]))  # string is truthy
print(any([None, False, 0, "", [], {}]))  # all falsy!
▶ Output
True ← found a 1
False ← no truthy values
False ⬅ 😲 No True found!
True ← "non-empty" string is truthy
False ← None, False, 0, "", [], {} are all falsy
all_demo.py
print(all([1, 2, 3]))     # all truthy ✓
print(all([1, 0, 3]))     # 0 is falsy!
print(all([]))           # ← empty list??
▶ Output
True ← all truthy
False ← 0 is falsy
True ⬅ 🤯 Vacuously true!
😊 any() — The Optimist
Returns True if at least one item is truthy.
🎯 all() — The Perfectionist
Returns True only if every item is truthy.
📋 Quick Reference
CallResultWhy
any([0,0,1])TrueFound truthy
any([])FalseNone found
all([1,2,3])TrueAll truthy
all([])True ⚠️No falsy found
DEMO 03

The Workhorses: sum · len · max · min

Clean, readable math — no manual loops needed
workhorses.py
scores = [85, 92, 78, 95, 88, 61]
name   = "Python"

print("Count  :", len(scores))
print("Total  :", sum(scores))
print("Average:", sum(scores) / len(scores))
print("Highest:", max(scores))
print("Lowest :", min(scores))
print("Letters:", len(name))    # len works on strings too!
▶ Output
Count : 6
Total : 499
Average: 83.1666...
Highest: 95
Lowest : 61
Letters: 6 ← "Python" has 6 chars
✅ No Reinventing the Wheel
Before built-ins, you'd write a for loop to sum, another to find max/min. These do it in one readable line — no boilerplate needed.
⚡ len() Counts Everything
len() works on lists, strings, dicts, and tuples. Even len("hello world") counts the space — it's 11, not 10!
DEMO 04

The Functional Trio: zip · filter · map

Data processing at scale — building blocks of analysis
🤐 zip()
Pairs iterables like a zipper
names = ["Alice","Bob"]
scores= [95, 82]
pairs = list(zip(
    names, scores))
print(pairs)
▶ Output
[('Alice', 95),
('Bob', 82)]
🔍 filter()
Keep only matching items
nums = [1,2,3,4,5,6]
evens = list(filter(
    lambda x: x%2==0,
    nums))
print(evens)
▶ Output
[2, 4, 6] ← only evens
🔄 map()
Transform every item
temps_c = [0,20,37,100]
temps_f = list(map(
    lambda c: c*9/5+32,
    temps_c))
print(temps_f)
▶ Output
[32.0, 68.0,
98.6, 212.0]
⚡ Pro-tip: zip, filter, and map are lazy — they return iterators, not lists. Wrap in list() to materialize the results. This saves memory on large datasets.

DEMO 06 — LIVE RACE 🏎️

The Speed Race: tuple vs frozenset

O(n) linear search vs O(1) hash lookup — watch the difference
📋 tuple — Linear Search O(n)
race_tuple.py
import timeit
t = tuple(range(1_000_000))
time = timeit.timeit(
  lambda: 999_999 in t, number=100)
print(f"Tuple: {time:.4f}s")
🧊 frozenset — Hash Lookup O(1)
race_frozenset.py
import timeit
fs = frozenset(range(1_000_000))
time = timeit.timeit(
  lambda: 999_999 in fs, number=100)
print(f"frozenset: {time:.6f}s")
tuple (1M items)
frozenset (1M items)
▶ Simulated Result
tuple : ~0.0420s ← checked 999,999 items one by one
frozenset: ~0.0000023s ← direct hash lookup
Speed diff: frozenset is ~18,000× faster 🚀
🔑 Key Difference
frozenset can be used as a Dictionary Key — a regular set cannot (it's unhashable). Use frozenset when you need an immutable, fast-lookup collection as a dict key.
DEMO 07

frozenset as a Dictionary Key

The use case that sets frozenset apart from tuple
friendship_tracker.py
# AI friendship tracker
# Order shouldn't matter!

friends = {}

# Using tuple as key:
friends[("Alice", "Bob")] = "close"
print(friends.get(("Bob", "Alice")))

# Using frozenset as key:
friends2 = {}
friends2[frozenset(["Alice","Bob"])] = "close"
key = frozenset(["Bob", "Alice"])
print(friends2.get(key))
▶ Output
None ← tuple failed! ("Bob","Alice") ≠ ("Alice","Bob")
'close' ← frozenset succeeded! Order doesn't matter

🧊 Think of it like this:

tuple = printed grocery list (order matters)
frozenset = locked glass jar of marbles (what's inside matters, not arrangement)
